# RuTracker Архив — Поиск и просмотр торрентов

Полная база всех раздач RuTracker у вас на компьютере. Офлайн-поиск по миллионам торрентов, просмотр описаний с BBCode и стриминг медиа-файлов через [Webtor](http://localhost:8082).

## Быстрый старт (Docker)

### 1. Подготовьте архив

Магнет-ссылка для загрузки полной базы RuTracker:

```
magnet:?xt=urn:btih:ff805117d5e6258ba71a21bdb9a322cbe0338fa0&dn=rutracker-20260329.xml.xz&tr=http%3A%2F%2Fbt4.t-ru.org%2Fann%3Fmagnet
```

Скачанный файл `rutracker-20260329.xml.xz` положите в корень проекта.

### 2. Запустите PostgreSQL

```bash
make docker-pg
```

### 3. Загрузите данные

```bash
make docker-migrate
```

Миграция занимает ~1-2 часа. Прогресс: `make docker-logs-migrator`.
При обрыве — просто перезапустите, миграция продолжится с места остановки.

### 4. Запустите сервер

```bash
make docker-server
```

Откройте **http://localhost:8080** для поиска торрентов.

## Порядок запуска

Важно соблюдать последовательность:

```bash
make docker-pg        # 1. Поднять базу данных
make docker-migrate   # 2. Загрузить данные (дождаться окончания)
make docker-server    # 3. Запустить веб-сервер
```

Остановить: `make docker-down`
Очистить всё: `make docker-down-v`

## Возможности

### Поиск и навигация

- **Полнотекстовый поиск** по заголовкам через `websearch_to_tsquery('russian', ...)`
- **Фильтрация по категориям** — каскадные dropdown: Категория → Подкатегория → Подподкатегория
- **Сортировка** — по дате или размеру раздачи
- **Пагинация** — быстрая, без дорогого `COUNT(*)`

### Страница торрента

- **Рендер BBCode** — спойлеры, цитаты, картинки, шрифты, цвета, смайлики (27 видов)
- **Список файлов** — директории и файлы с размерами
- **Магнет-ссылки** — корректная генерация с трекером RuTracker
- **Старые версии** и дубликаты раздач

### 🎬 Стриминг через Webtor

На каждой странице торрента есть ссылка **«Обзор торрента»** → открывает раздачу в [Webtor](http://localhost:8082).

**Webtor** — self-hosted сервис для стриминга торрентов прямо в браузере:

- Скачивает файлы по мере воспроизведения (не нужно ждать полной загрузки)
- Поддерживает видео и аудио в любом формате (транскодинг на лету)
- Позволяет выбрать конкретный файл из раздачи
- Работает через BitTorrent P2P с публичными трекерами

> **Как это работает:** Нажимаете ссылку → Webtor получает magnet-ссылку → подключается к пирам → начинает скачивать и одновременно воспроизводить файл.

#### ⚠️ Важно знать

**Скачивание и раздача:** Webtor работает как BitTorrent-клиент — скачивает нужный файл и раздаёт его другим пирам. Пока контейнер запущен, раздача продолжается.

**Расход трафика:** Так как это P2P, Webtor отдаёт скачанные файлы другим пользователям. Если переживаете за трафик — останавливайте контейнер:
```bash
docker compose stop webtor   # Остановить
docker compose start webtor  # Запустить снова
```

**Место на диске:** Скачанные файлы хранятся в кэше. Webtor **автоматически чистит** старые файлы когда заканчивается место. Можно настроить пороги:
```yaml
environment:
  - CLEANER_FREE=30%       # Сколько освобождать
  - CLEANER_KEEP_FREE=50G  # Начинать чистку когда свободно < 50G
```

**Итог:** Webtor безопасен для постоянного использования благодаря автоочистке, но если не пользуетесь — лучше останавливать.

## API

```bash
# Поиск
GET /api/search?q=фильм&cat=Кино&limit=20&offset=0&sort=date

# Получить торрент
GET /api/torrent/123456

# Список категорий
GET /api/categories

# Подкатегории категории
GET /api/subcategories?cat=Игры

# Подподкатегории
GET /api/subsubcategories?cat=Apple&sub=iOS
```

Ответы в формате JSON. Все эндпоинты доступны без аутентификации.

## Локальная разработка (без Docker)

```bash
# Зависимости
go mod download

# Сборка
make build

# Запуск сервера (требует PostgreSQL на порту 5433)
make run-server

make run-migrator
```

## Сервисы Docker

| Сервис | Порт | Описание |
|---|---|---|
| `postgres` | 5433 → 5432 | PostgreSQL 16 с полнотекстовым поиском |
| `migrator` | — | CLI для загрузки XML архива в БД |
| `server` | 8080 | Веб-сервер с поиском и просмотром |
| `webtor` | 8082, 6881 | Self-hosted стриминг торрентов (BitTorrent P2P) |

## Технологии

- **Go 1.21+** — стандартная библиотека `net/http`, `html/template`
- **PostgreSQL** — полнотекстовый поиск через `tsvector` + GIN индекс
- **pgx/v5** — драйвер для PostgreSQL
- **BBCode** — кастомный парсер для RuTracker формата
- **Webtor** — self-hosted стриминг торрентов (отдельный сервис)
